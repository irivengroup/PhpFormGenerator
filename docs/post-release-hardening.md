# Post-release Hardening

## Objectif
Documenter la passe de stabilisation réalisée juste après la sortie V5.0.0.

## Zones contrôlées
- chaîne plugin/runtime
- compatibilité des registres
- accessors publics de `FormPluginKernel`
- cohérence du SDK frontend
- non-régression de l’API publique figée
